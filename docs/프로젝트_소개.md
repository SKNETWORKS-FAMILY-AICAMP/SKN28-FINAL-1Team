# AI 개인화 패션 추천 서비스 — 아이디어 소개 및 계획

## 1. 서비스 모델 구현 목표 · 개발 내용 · 개발 범위

### 아이디어 소개

본 서비스는 사용자가 보유한 옷장을 디지털로 등록하고, 이를 기반으로 개인화된 패션 추천을 제공하는 AI 시스템이다. 기존 패션 추천 서비스가 상품 판매 중심인 것과 달리, "내가 이미 가진 옷"을 출발점으로 삼아 실질적인 착장 문제를 해결하는 것이 핵심 아이디어다.

### 구현 목표

서비스는 세 가지 AI 기능을 목표로 한다. 첫째, **옷장 내 코디 추천**은 사용자가 등록한 의복 아이템을 입력으로 받아 오늘의 날씨와 TPO(시간·장소·상황)를 고려한 착장 조합을 RAG 기반으로 추천한다. 둘째, **신규 아이템 추천**은 보유 의복 아이템과 체형 정보를 함께 고려하여 옷장과 어울리는 새로운 의상 아이템을 RAG 기반으로 추천한다. 셋째, **사이즈 추천**은 사용자의 체형 정보를 아이템 실측 데이터와 매칭하여 적합한 의복 사이즈를 제시한다. 시스템 전반의 핵심 가치는 추천 정확도, 응답 속도, 확장 가능한 배포 구조다.

### 개발 내용 및 범위

프론트엔드는 모바일 애플리케이션으로 구현하며, 옷장 아이템 등록, 체형 정보 입력, 추천 요청 및 결과 조회 기능을 담당한다. 백엔드는 두 개의 서버로 분리한다. 인증 서버는 회원가입·로그인과 JWT 발급·검증을 전담하는 독립 서버로 구현하고, API 서버(Django/DRF)는 옷장·사용자·체형 정보 관리와 추천 요청 오케스트레이션을 담당한다. AI 계층은 위 세 가지 추천 모듈(RAG 2종 + 사이즈 추천)로 구성하며, 웹 계층과 분리된 ml/ 모듈로 개발한다. 모델 학습·실험은 RunPod GPU 환경(임시)에서 수행하되, 최종 서비스는 AWS(EC2/ECS, RDS, S3) 프로덕션 배포를 전제로 이식성을 유지한다.

## 2. 시스템 구성 및 아키텍처

시스템은 클라이언트 — 백엔드 — AI 추천 엔진 — 데이터 계층의 4계층과 외부 서비스로 구성된다.

모바일 앱은 인증 서버에서 JWT를 발급받은 뒤, 토큰을 포함해 API 서버에 추천을 요청한다. API 서버는 토큰을 검증하고 요청을 AI 추천 엔진의 각 모듈로 라우팅한다. ① 코디 추천 RAG는 외부 날씨 API로 당일 날씨를 조회하고 TPO 컨텍스트와 결합해 옷장 내 코디를 생성하며, ② 신규 아이템 추천 RAG는 Vector DB의 상품 임베딩 유사도 검색으로 후보를 찾은 뒤 체형 정보를 반영해 추천한다. 두 RAG 모듈은 LLM API를 호출해 최종 추천 문장을 생성한다. ③ 사이즈 추천은 체형 정보와 아이템 실측 데이터를 매칭한다.

데이터 계층은 PostgreSQL(RDS)이 사용자·옷장·체형 데이터를, Vector DB가 의류·상품 임베딩 검색 인덱스를, Redis가 추천 결과와 날씨 캐시를, S3가 의류 이미지와 메타 데이터를 담당한다. 추천 결과는 Redis에 캐싱되어 응답 속도를 확보한다.

## 3. 주요 특징 및 핵심 기술

### 아이디어 컨셉

"쇼핑몰이 팔고 싶은 옷"이 아니라 "내 옷장에서 오늘 입을 수 있는 옷"을 추천하는 옷장 중심(closet-first) 컨셉이다. 사용자의 실제 보유 아이템을 데이터의 출발점으로 삼기 때문에 추천이 곧바로 실행 가능하며, 새 옷 추천 역시 기존 옷장과의 조합 가능성을 근거로 제시되어 불필요한 소비 대신 활용도 높은 구매를 유도한다.

### 핵심 기술

첫째, **RAG(Retrieval-Augmented Generation) 기반 추천**이다. 의류 아이템을 임베딩으로 변환해 Vector DB에 색인하고, 추천 시점에 옷장·날씨·TPO·체형 등 사용자 컨텍스트로 후보를 검색한 뒤 LLM이 근거를 갖춘 추천 결과를 생성한다. 단순 협업 필터링과 달리 "왜 이 조합인지"를 자연어로 설명할 수 있는 것이 강점이다. 둘째, **실시간 컨텍스트 결합**으로, 날씨 API와 TPO 입력을 추천 파이프라인에 주입해 같은 옷장이라도 상황마다 다른 결과를 제공한다. 셋째, **체형 기반 개인화**로, 체형 정보를 신규 아이템 추천과 사이즈 추천 양쪽에 활용해 "어울리는 옷"과 "맞는 사이즈"를 함께 제시한다. 넷째, **서비스 분리 아키텍처**로, 인증 서버·API 서버·ML 계층을 분리해 각 컴포넌트를 독립적으로 확장·배포할 수 있으며, Redis 캐싱으로 추천 응답 속도를 확보한다.

### 활용성 및 특징

일상 사용자의 "오늘 뭐 입지" 문제를 해결하는 데일리 유틸리티로 활용되며, 여행·면접·경조사 등 TPO 기반 상황별 코디 제안으로 확장 가능하다. 옷장 데이터가 축적될수록 추천 품질이 향상되는 데이터 선순환 구조를 가지며, 신규 아이템 추천은 커머스 연계(제휴·광고) 수익 모델로 발전할 수 있다. 기술적으로는 RunPod GPU에서 학습한 모델을 환경변수 기반 추상화를 통해 AWS 프로덕션으로 그대로 이관할 수 있는 이식성을 갖추어, 비용 효율적인 개발과 안정적인 운영을 동시에 달성한다.

## 4. 기대효과 및 활용 방안

### 파급효과와 첨단·디지털 신기술 활용도

본 모델이 실제 구현되면 패션 추천의 기준이 "판매자 재고"에서 "사용자 옷장"으로 이동하는 파급효과가 기대된다. 사용자는 보유 의류의 활용률을 높여 중복 구매를 줄이고, 커머스 측은 옷장 데이터에 근거한 높은 구매 전환율의 추천 채널을 확보한다. 기술 활용 측면에서는 학습한 신기술이 전 구간에 적용된다. LLM과 RAG는 임베딩 검색(Vector DB)과 결합해 근거 있는 생성형 추천을 구현하고, PyTorch 기반 딥러닝은 의류 이미지 임베딩과 체형-사이즈 매칭 모델에 활용된다. 클라우드 네이티브 기술(AWS ECS/RDS/S3, Redis 캐싱, Docker 컨테이너화)과 GPU 클라우드(RunPod)를 통해 MLOps 전주기를 실습·적용하며, JWT 기반 인증 분리 등 확장 가능한 백엔드 설계 역량도 함께 검증한다.

### 현실적 구현 가능성과 실제 적용 방안

본 아이디어는 검증된 기술 스택의 조합으로 구성되어 구현 가능성이 높다. RAG 파이프라인은 상용 LLM API와 오픈소스 Vector DB로 즉시 구축 가능하고, 날씨 데이터는 공공·상용 API로 확보되며, Django/DRF와 모바일 앱은 표준적인 개발 범위다. 가장 큰 부담인 GPU 비용은 RunPod 임시 환경으로 학습을 수행하고 추론만 AWS로 이관하는 전략으로 최소화한다. 실제 적용 시에는 개인 사용자용 데일리 코디 앱을 기본으로, 의류 커머스의 개인화 추천 위젯, 온라인 쇼핑의 사이즈 실패(반품) 감소 솔루션, 스타일리스트·리셀 플랫폼의 옷장 관리 도구 등 B2B2C 형태로 확장 적용할 수 있다.

### 향후 발전 가능성과 사회적 기여도

향후에는 착장 이미지 기반 가상 피팅(Virtual Try-On), 계절·트렌드를 반영한 옷장 리포트, 사용자 간 스타일 커뮤니티 등으로 발전할 수 있으며, 축적된 옷장·체형 데이터는 패션 수요 예측 등 데이터 사업으로 확장 가능하다. 사회적으로는 보유 의류 활용을 극대화해 충동 구매와 의류 폐기물을 줄이는 지속가능한 소비 문화에 기여하고, 사이즈 추천을 통한 반품 감소는 물류 비용과 탄소 배출 절감으로 이어진다. 또한 체형 기반 추천은 표준 체형 중심의 기성 패션 정보에서 소외되던 다양한 체형의 사용자에게 맞춤 정보를 제공한다는 점에서 포용적 가치를 갖는다.
