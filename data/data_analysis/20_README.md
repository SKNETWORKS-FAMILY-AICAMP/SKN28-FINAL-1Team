# 20. 한국인 전신 형상 및 치수 측정 데이터 보는 법

## 1. 이 데이터는 무엇인가

`s3://skn28-cozy/20.한국인_전신_형상_및_치수_측정_데이터/`는 한국인 성인 약 1,000명(여 500 + 남 500)을 정해진 자세로 다방면 촬영하고, 동시에 **37개 신체 치수(키·몸무게·각 부위 둘레·길이·너비)**를 실측한 **의류 치수 설계용 정밀 인체 데이터**다. 폴리곤 어노테이션이 달린 2D 전신 사진, PLY 형식의 3D 형상 메쉬, 그리고 1인 1행짜리 CSV의 실측치 표가 한 묶음으로 묶여 있다.

핵심 자산:

- **신체 실측치 CSV** (1인 1파일): 키, 둘레(머리/목/가슴/허리/엉덩이/넙다리/무릎/장딴지/종아리/발목/위팔/팔꿈치/손목 13종), 길이(목뒤/엉덩이/겨드랑/허리/샅/무릎 6종), 너비(어깨사이/발/얼굴 3종), 팔 길이·둘레 4종, 머리·얼굴·손·발 디테일 7종, 신발 굽 높이 1종, 신장·몸무게·체지방율·성별·나이 → **합 37개 측정값**.
- **2D 전신 사진 + 폴리곤 어노테이션** (1인 ≈ 1,128장): 36개 측정 컨텍스트(6×6 매트릭스) × 다수 신체 부위 마스크(머리, 몸통, 어깨, 좌/우 팔꿈치, 손목, 허벅지, 좌/우 종아리, 발목 등). 어노테이션은 픽셀 좌표의 `polygon` 형식.
- **3D 형상 메쉬** (1인 1쌍): PLY(binary, vertex 272,576개 + face 545,174개, 텍스처 좌표 포함, ≈ 27MB) + MTL(208 바이트 재질 정의). Blender/MeshLab에서 바로 렌더링 가능.
- **추가 데이터(230524add / 230704add)**: 2023-05-24(Training)·2023-07-04(Validation)에 확장된 인물 풀(같은 형식).

**우리 프로젝트 활용 가치**: 폴리 어노테이션 사진은 "신체 부위별 segmentation 학습 데이터"로, 3D 메쉬는 "체형 가상 피팅" 프로토타입 데이터로, CSV는 "체형 군집화·사이즈 매칭의 정답지"로 동시에 쓸 수 있다. 10번 데이터(3D keypoint + 사진)가 "포즈 다양성" 위주인 반면, 20번은 "실측 치수 + segmentation" 위주.

## 5. 가장 중요한 파일: F009.csv 한 줄

이 데이터의 정체성은 "977+1,000명 × 37개 실측치 = 약 73,000개 셀짜리 표"다. 그 본질은 **한 사람에 대한 한 행 = 한 행의 실측치 37개**이며, 이 37개 셀이 곧 "이 사람의 의류 사이즈 프로필"이다.

왜 csv가 가장 중요한가:

- **모든 권위 있는 정보의 출처**: 사진·메쉬는 "시각화/검증용"이고, 실제 의류 치수 설계·체형 군집화·사이즈 매칭은 모두 이 37개 실측치에서 시작한다.
- **1행 = 1인 단위 row**: 다른 어떤 포맷(JSON, PLY)도 한 사람에 대해 한 행짜리 표로 평탄화 가능. pandas로 바로 적재.
- **컬럼명이 표준 의류 치수 사전**: "어깨사이너비·배꼽수준허리둘레·종아리최소둘레" 등은 의류 패턴 설계의 표준 항목이며, 한국 의류 사이즈 표준(KS K 0051)을 따름.
- **컬럼명이 표준 의류 치수 사전**: "어깨사이너비·배꼽수준허리둘레·종아리최소둘레" 등은 의류 패턴 설계의 표준 항목이며, 한국 의류 사이즈 표준(KS K 0051)을 따름.

## 8. 이 데이터로 알 수 있는 것

각 항목에 대해 **무엇을 알 수 있는가** + **왜 그렇게 판단하는가(근거)** 형태로 정리.

### 8.1 한국인 성인 1,000명의 37개 신체 치수 실측 분포

- **무엇**: 키·몸무게·체지방율, 13개 둘레, 6개 신장 길이, 2개 신분 너비, 팔 관련 5개, 머리·얼굴 3개, 발·손 4개, 메타 4개
- **왜 그렇게 판단**: csv 행 2의 37개 컬럼이 정확히 이 범주. F009 실측치(키 161.1, 허리 75.9, 엉덩이 97.8)가 한국 여성 40대 평균의 직관적 범위에 들어와 데이터 신뢰성 확인
- **활용**: 한국 의류 사이즈 표준(KS K 0051) 갱신, 패턴 생성기 입력값, 사이즈 매칭 룰 학습

### 8.2 성별 × 연령대 × BMI대별 체형 군집

- **무엇**: 약 500+500 = 1,000명 실측 분포에서 자연스러운 클러스터(예: "20대 마른 여성 / 40대 중간 남성 / 60대 비만 여성") 추출
- **왜 그렇게 판단**: 1,000명 표본이면 성별(2) × 연령대(5) × BMI 4분위 = 40개 셀로 나눠도 셀이 25명씩 → 군집 평균·분산 안정적 추정 가능
- **활용**: 사용자 입력 정보 → "가장 가까운 체형 군집" → 그 군집 평균 사이즈/핏 추천. cold-start 문제의 정답 데이터

### 8.3 신체 부위별 segmentation (머리/몸통/팔/다리 등)

- **무엇**: 사진 한 장에서 어떤 픽셀이 머리/몸통/팔/종아리/발목인지 픽셀 단위 라벨

- **활용**: "사용자 옷장 사진" 자동 부위 segmentation 모델의 학습·검증. 가상 피팅·체형 분석의 기반

### 8.4 3D 전신 메쉬 (vertex 27만 / face 54만 / 텍스처 포함)

- **무엇**: 1mm 해상도 수준의 3D 신체를 위·법선·텍스처 좌표와 함께 얻음
- **왜 그렇게 판단**: `F001_3D.ply` 헤더가 `vertex 272576, face 545174, property list uchar float texcoord`로 텍스처 좌표까지 포함된 완전한 메쉬임을 명시
- **활용**: AR 가상 피팅, 신발/모자 등 부분 의류 3D 매칭, 의류 패턴 3D 시뮬레이션 입력

### 8.5 다각도·다 컨텍스트 촬영 (1인 ≈ 1,128장)

- **무엇**: 36개(6×6) 측정 컨텍스트에서 같은 사람을 다른 각도/조건으로 본 사진 + annotation. 컨텍스트 04는 키포인트 28개, 나머지는 32개

### 8.6 촬영 일시·카메라 위치·자세·의상 메타

- **무엇**: 2021년 8월 촬영(F009의 `2021:08:24 12:33:26`), 카메라 위치(1~N), 자세·의상 코드(`position`, `cloth` 필드)

### 8.7 표준 의류 치수와 한국인 실측의 차이

- **무엇**: 표준 사이즈 표(85-90-95 등)와 1,000명 실측의 분포 비교 → "표준이 실제 한국인 분포의 어디에 위치하는지" 정량화

- **활용**: "이 사용자 체형이면 95 사이즈가 표준이지만 한국인 데이터로는 100에 가까움" 같은 정밀 추천

- **무엇**: 검수에서 반려된 샘플은 `inspRejectYn=Y`로 즉시 식별

- **활용**: 학습 데이터 품질 관리의 정식 필터. `Y` 샘플은 별도 분석(왜 반려됐는지)

## 9. 추천 시스템에 활용하는 방법

### 9.1 사용자 입력 → 체형 군집 매칭 (cold-start 해결)

CSV 37개 컬럼을 모두 활용하면 "사용자가 입력한 키·몸무게·나이·성별"만으로 1,000명 실측 분포에서 가장 가까운 군집을 찾을 수 있다. 이 군집의 평균 어깨너비·가슴둘레·엉덩이둘레를 추천 로직에 사용.

### 9.2 사이즈 매칭 룰 학습의 정답 데이터

상품별 사이즈 표(가슴단면·허리단면·총장)와 사용자 실측 37개 값을 매칭해 "이 사용자 체형이면 M보다 L이 맞다" 같은 룰을 학습. 10번 데이터(3D keypoint)보다 더 많은 측정 항목을 단일 actor에 제공.

### 9.3 segmentation 모델 학습·검증

`json/`의 폴리곤 annotation은 머리·몸통·팔·다리 등 부위별 segmentation 모델의 학습·검증 데이터. "사용자가 올린 옷장 사진"의 부위 자동 인식에 활용. 1,128장/인물 × 1,000명 = 약 113만 장의 대규모 데이터.

### 9.4 가상 피팅 / 체형 시각화 프로토타입

`원천데이터/`의 PLY 메쉬는 AR 가상 피팅의 3D 신체 모델로 활용. 텍스처 포함이라 컬러도 매핑 가능. 10번의 3D 데이터(여러 포즈)와 달리 20번은 "실측 신체 1개"에 집중.

### 9.5 10번·20번 데이터의 결합 활용

10번(3D keypoint + 다 포즈 + 사진)과 20번(37개 실측치 + segmentation + 3D 메쉬)을 같은 actor_id(F009, M001 등)로 매칭 가능. 10번의 32개 keypoint를 20번의 37개 실측치와 매핑하면 "keypoint 위치 → 둘레/길이" 역산 모델 학습.

### 9.6 TPO/스타일 → 부위 강조 추천

체형 군집별로 "어깨가 넓은 사용자" / "허리가 굵은 사용자" 같은 특성을 추출하고, 특정 의류(오버사이즈 자켓, 와이드 팬츠)가 잘 어울리는 체형 조건을 학습 데이터로 활용.

### 9.7 트렌드·시장 분석 데이터

`측정월·측정일` 메타로 2021~2023년 한국인 체형 기준선을 시간순 분석. 향후 동일 프로토콜로 재촬영 시 체형 변화 추적.

**주의사항**:

- **CSV 인코딩**: EUC-KR. UTF-8로 열면 한글이 전부 깨진다. Python `open(path, encoding='cp949')`, pandas `read_csv(path, encoding='cp949')`.
- **decimal 콤마 이슈**: Validation의 일부 행에서 `"98,6"`처럼 콤마가 소수점 대신 쓰인 사례 있음(F001 행 2의 14번 컬럼). 한국 로케일 CSV 파싱 시 `decimal=','` 옵션 또는 후처리 필요.
- **1,128개 JSON의 컨텍스트 차이**: 6×6 = 36 컨텍스트 중 컨텍스트 04만 ZZ가 05~32(28개)라서 32개가 아님. 모든 컨텍스트가 동일하다는 가정으로 처리하면 4×4=16개 샘플 미스.
- **3D mesh 포맷 2종**: 원천데이터/TS_3D_data는 **OBJ** 포맷 (≈ 49MB/명), 라벨링데이터_230524add PLY는 **binary PLY** (≈ 27MB/명). OBJ는 텍스트라 로드 쉽지만 파일 크기 더 큼.
- **2D 사진 위치**: JPG는 **원천데이터/TS_*/Image/**에 있음. 라벨링데이터/TL_*/에는 JSON만 존재.
- **CSV 2곳 중복**: 실측치 CSV는 라벨링데이터/TL_*/csv/와 원천데이터/TS_*/csv/ 두 곳에 동일 내용으로 중복 존재.
- **map_Kd 텍스처 경로**: MTL의 `map_Kd`는 상대 경로. S3에서 받은 후엔 같은 디렉토리에 텍스처 JPG를 함께 두어야 함.
- **position·cloth 코드 미확인**: json의 `position`, `cloth` 값은 라벨링된 코드로 추정되나, 코드→의미 매핑 사전은 본 단계에서 미확인.
- **라벨링데이터_230524add**: PLY만 있고 JSON/CSV 없음. 1,000명 (F001~F1000 추정). 원래 977명과 중복 여부는 미확인.
