# 23 · 공공데이터 의류생활체육 추천 데이터

## §1. 데이터 한 줄 요약

- **누가 만들었나**: 한국콘텐츠진흥원(KOCCA) + 한국정보통신기술협회(TTA)가 공공데이터로 개방한 **의류·생활체육 추천 학습 데이터**.
- **무엇이 있나**: 남성 모델 전신 사진 10,000장 이상 + 권장 스타일 라벨(bold/hippie/hiphop).
- **왜 추천 시스템에 유용한가**: 사람 사진 + 스타일 태깅이 함께 있어서 **이미지 기반 스타일 분류/추천 모델** 학습 데이터로 즉시 활용 가능.

## §2. 이 데이터로 알 수 있는 것 (체크리스트)

- ✅ 성별 + 연령대(출생년도 코호트)별 선호 스타일이 다른지
- ✅ 특정 스타일(hippie/bold/hiphop)에 어울리는 상의·하의·신발·액세서리 조합이 무엇인지
- ✅ 남성 패션에서 같은 아이템이라도 어떤 코디가 "추천 점수"가 높은지
- ✅ 사진 한 장만 보고 스타일 카테고리를 자동 태깅하는 모델 학습 근거
- ❌ 실제 사용자별 선호도 (개인화 추천) — user-item interaction 데이터는 없음
- ❌ 여성/아동 데이터 — 현재 남성 데이터만 확인됨 (전체 개방 범위 확인 필요)

## §3. 이 데이터로 알 수 없는 것

- 컬러 팔레트 / RGB 통계 — 라벨 없음
- 브랜드 정보 — 사진만 있고 브랜드 메타 없음
- 가격대 / 구매 의도 — 추천 점수만 있을 뿐 상업 데이터 없음
- 시간·계절·TPO(출근/면접/여행 등) 라벨 — 없음
- 사용자 만족도/리뷰 텍스트 — 정답지는 "이 스타일인가" 하나뿐

## §6. 샘플 하나를 보는 순서 (왜 이렇게 보는가)

`T_16774_70_hippie_M.jpg` 한 장을 열었을 때 추출할 정보를 단계별로:

1. **파일명에서 정답 라벨 읽기** — `T_<ID>_<연도코드>_<스타일>_<성별>` 형식이라 파일명만 봐도 라벨을 알 수 있다.
   *왜?* 학습 시 라벨 CSV 없이 폴더 구조만으로 supervised 학습을 할 수 있게 해준다 — **이미지 분류 모델 baseline**으로 적합.
2. **메타데이터 확인** — `file` 명령으로 `JPEG image data, Exif standard` 확인. 3000×4000 고해상도.
   *왜?* 해상도가 높다는 건 입력 이미지 augmentation(회전/크롭/색감) 여지가 충분하다는 뜻.
3. **이미지 시각적 내용 확인** — 남자 모델이 데님 자켓+데님 팬츠 코디(denim on denim), 흰 운동화, 모자. 배경은 붉은 벽돌.
   *왜?* 사람이 "hippie 스타일은 이렇게 생겼다"라고 라벨링할 때 참조하는 시각적 특징을 학습 데이터에 포함시키기 위해.
4. **메타에 없는 정보 기록** — 색상 팔레트(블루 데님 / 화이트 / 회색 벽돌), 아이템 구성(자켓+팬츠+운동화+모자).
   *왜?* 라벨은 "hippie" 하나뿐이지만 추천 시스템에 쓰려면 색상/아이템 레이어까지 분해해 매핑해야 한다. 이 한 장이 "denim on denim + sneakers + cap" 조합의 실증 사례가 된다.

## §8. 이 데이터로 알 수 있는 것 — 자동화 추천 시스템 관점

체크리스트 + 추론 이유를 한눈에:

| 활용 RAG | 입력 | 출력 | 데이터 근거 |
| --- | --- | --- | --- |
| **RAG 1 (체형/사이즈)** | ⚠️ 약함 | (신체치수 없음) | 사진만 있고 신체 측정값 CSV 없음 |
| **RAG 2 (패션 아이템)** | ✅ 매우 강함 | 스타일별 아이템 매핑 | 파일명에 명시된 `<style>` 태그 (hippie/bold/hiphop …) |
| **RAG 3 (TPO/스타일)** | ✅ 강함 | 남성 + 연령별 선호 스타일 | 코호트 폴더 (`TS_man_<year>`) |

판단 근거:
- **RAG 2 활용도 최고인 이유**: 파일명 자체에 카테고리형 라벨이 박혀 있어서 라벨링 비용 0으로 학습 데이터 투입 가능.
- **RAG 3 활용 이유**: 폴더 구조가 (성별 × 출생연도)로 나뉘어 있어 demographic별 분포 시각화가 자연스럽다.
- **RAG 1 활용 부재 이유**: 키/몸무게/사이즈 같은 정량 데이터가 없어 가상 피팅 모델에는 직결 불가. 단, 사진 속 실루엣을 추정해 추정 BMI를 만드는 2차 파이프라인은 가능.

## §9. 추천 시스템에 활용하는 방법

즉시 활용 가능한 파이프라인:

1. **이미지 분류 모델 baseline (CNN/CLIP fine-tune)**
   - 입력: JPG 3000×4000
   - 라벨: 파일명의 `<style>` (hippie/bold/hiphop)
   - 출력: 새 이미지 1장이 들어왔을 때 스타일 카테고리 + 신뢰도
   - 추천 시스템 연결: 후보 의류 이미지가 주어졌을 때 "사용자가 선호하는 hippie 점수"로 ranking score의 한 축.

2. **유사 이미지 검색 (벡터 임베딩)**
   - 같은 스타일 폴더 안의 1652장(예: TS_man_1970/hippie)을 임베딩 후 클러스터링.
   - 사용자 의류 이미지 1장 → 가까운 이웃 K개 → "비슷한 착장" 추천.

3. **스타일별 아이템 디테일 추출**
   - 한 장의 사진에서 (상의 종류, 하의 종류, 신발, 액세서리) 박스 단위로 crop → 폴더 단위로 분포 집계 → "hippie는 보통 X, 보통 Y다" 룰 생성.

## §10. 데이터 품질 / 한계

- ✅ 장점
  - 파일명 자체에 라벨이 있어 별도 어노테이션 불필요.
  - 해상도가 높아 모델 입력으로 충분.
  - 코호트 분리가 깔끔해서 train/val split이 자연스럽다.
- ⚠️ 한계
  - **여성·아동 데이터 부재**: 남성 위주. 만약 여성 사용자 대상 추천이면 별도 데이터 필요.
  - **스타일 3종 가량**: bold/hippie/hiphop 외 다른 스타일 데이터가 더 있는지 추가 확인 필요.
  - **시간 라벨 부재**: 사진이 몇 년도에 찍혔는지 모름. 트렌드 변화 학습 불가.
  - **단일 아이템 매핑 어려움**: 한 사진에 여러 아이템이 같이 있고 분할 라벨이 없어 아이템 단위 추천 룰 추출은 별도 모델 필요.

## §12. 결론 / 액션 아이템

1. **먼저 쓸 것**: 폴더 구조 그대로 Label map을 만들어서 CNN/CLIP 분류 baseline 학습. **RAG 2(아이템) + RAG 3(스타일)** 동시 보강.
2. **다음에 확장**: Validation 폴더 데이터도 받아 모델 일반화 성능 평가.
3. **보완이 필요한 것**: 컬러/아이템 디테일 라벨이 없어서 룰 추출이 약함 → 별도 detector(CNN/YOLO)로 사진 속 의상 박스 추출 후 자동 라벨링 파이프라인 권장.
4. **여성 의류 데이터로 확장 시**: 다른 데이터셋(예: 12_FASCODE Fashion-How 등)에서 보완 검토.
