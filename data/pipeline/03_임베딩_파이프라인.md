# 워크플로 3 — 임베딩 파이프라인

정제 완료된 이미지 키를 FashionSigLIP으로 임베딩해 Qdrant에 업서트하고, 포인트 수가 기대치와 맞는지 검증한다.

실물: `data/pipeline/03_workflow_embedding.json` (n8n 워크플로 ID: `WORKFLOW_ID_EMBED`)
노드 수: 8개

---

## 1. 무엇을 하는 워크플로인가

- 파이썬(`embed_pipeline.py`)이 S3 이미지를 내려받아 Marqo-FashionSigLIP으로 임베딩한다. [근거: scripts/embed_pipeline.py]
- Qdrant `fashion_items` 컬렉션에 업서트 후, HTTP로 포인트 수를 조회해 기대치와 대조.
- 일치하면 성공 알림, 불일치하면 경고 알림.

---

## 2. 노드 흐름도

```text
[Webhook Trigger]              webhook, POST /embed
      │
      ▼
[Prepare Embedding Batch]      code (JS)
      │                        입력 키에서 이미지 확장자만 필터
      │                        → /tmp/embed_keys.json 기록, 기대 건수(image_keys) 산출
      ▼
[Run Embedding Pipeline]       executeCommand
      │                        python3 .../embed_pipeline.py
      │                        --keys /tmp/embed_keys.json --collection fashion_items --batch-size 32
      ├───────────────┬────────────────────────────┐
      ▼               ▼                            (병렬 2갈래)
[Verify Qdrant    [Parse Embedding Results]     code (JS)
 Collection]         │                          stdout 파싱 → success/failed 집계
 httpRequest         │                          기대치와 비교, mismatch 판정
 GET .../collections/fashion_items
                     ▼
              [If Count Matches]                if  (mismatch == false?)
                     ├─ true  ──▶ [Slack: Embedding Success]         slack (continueOnFail)
                     └─ false ──▶ [Slack: Count Mismatch Warning]    slack (continueOnFail)
```

노드 8개: Webhook Trigger, Prepare Embedding Batch, Run Embedding Pipeline, Verify Qdrant Collection, Parse Embedding Results, If Count Matches, Slack: Embedding Success, Slack: Count Mismatch Warning. [근거: workflow_03_embedding.json]

참고: Run Embedding Pipeline 출력은 Verify Qdrant Collection과 Parse Embedding Results 두 노드로 동시에 갈라진다. IF 판정은 Parse Embedding Results 쪽 결과를 사용한다. [근거: workflow_03_embedding.json connections]

---

## 3. 왜 이렇게 설계했는가

- **임베딩 연산은 절대 n8n 노드로 하지 않는다.** 이미지 다운로드, FashionSigLIP 추론(GPU/CPU 무거운 연산)은 전부 `embed_pipeline.py` 안에서 돈다. n8n은 키 목록을 파일로 넘기고 한 번만 호출한다. [근거: n8n/README.md §정직한 평가, scripts/embed_pipeline.py]
- **결과 검증도 파이썬 출력 대조.** Code 노드는 스크립트 stdout(JSON `{total, success, failed}`)을 파싱해 기대 건수와 비교만 한다. [근거: Parse Embedding Results jsCode]
- **이중 검증.** 스크립트 출력(success 수)과 Qdrant 실제 포인트 수(HTTP 조회)를 둘 다 확인해, 임베딩은 됐는데 업서트가 누락된 경우를 잡는다. [근거: Verify Qdrant Collection httpRequest, If Count Matches]

## 4. 실행 방법

| 방식 | 내용 |
|---|---|
| 자동 (트리거) | 워크플로 2의 Trigger Embedding Workflow가 호출 (executeWorkflow). Webhook `POST /embed`도 존재 |
| 수동 | 각 노드를 Execute step으로 확인 |

전제 (2개 미충족):
1. `AWS_SECRET_ACCESS_KEY` 미등록 → S3 이미지 다운로드 실패.
2. 실제 임베딩에는 `marqo`/`torch`/`transformers`/`Pillow`/`qdrant-client`와 모델 실행 환경(GPU 워커)이 필요하다. **현재 경량 n8n 이미지에는 이 의존성을 넣지 않았다.** GPU 워커 연결 전까지는 워크플로 1·2까지만 실행 검증한다. [근거: n8n/README.md §5, scripts/embed_pipeline.py Dependencies]

Qdrant 접속: `QDRANT_URL` 환경변수 (기본 `http://host.docker.internal:6333`). [근거: docker-compose.yml, workflow JSON httpRequest url]
