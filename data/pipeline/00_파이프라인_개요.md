# 데이터 파이프라인 개요

S3 패션 데이터셋을 스캔 → 정제 → 임베딩하는 3단계 파이프라인을 n8n으로 오케스트레이션한다.
무거운 처리는 전부 파이썬이 하고, n8n은 트리거·순서·알림만 담당한다.

작성: 2026-07-13, 담당: 전하영(데이터)

## 2. 왜 n8n인가 — n8n은 오케스트레이션만

핵심 원칙: **무거운 데이터 처리를 n8n 노드로 직접 하지 않는다.**

- 수십 GB 이미지 다운로드, 임베딩 연산을 n8n Code/S3 노드로 돌리면 메모리·타임아웃·재시도 지옥이 된다. [근거: n8n/README.md §정직한 평가]
- n8n은 "언제 무엇을 실행할지"(트리거·순서)와 결과 "알림"만 정한다.
- 실제 처리는 `python3 /repo/...` 파이썬 스크립트가 한다. Execute Command 노드가 컨테이너 안에서 스크립트를 호출하는 구조다. [근거: workflow JSON의 executeCommand 노드]
- 워크플로 3개 모두 이 구조를 따른다: n8n 노드는 얇게, 파이썬이 두껍게.

언제 n8n이 필요한가:
- 지금 같은 탐색적 1회성 분석에는 n8n 없이 스크립트 수동 실행이 더 빠르다.
- 신규 데이터 주기 감지, 정제→임베딩→업서트가 정례화되는 "반복 배치" 시점부터 가치가 생긴다. [근거: n8n/README.md §정직한 평가]

## 4. 실물 파일 위치 (중요)

문서는 `data/pipeline/`에 있지만, **실물(워크플로 JSON·스크립트·docker 파일)은 `data/pipeline/`에 있다.**

| 종류 | 경로 |
|---|---|
| 워크플로 JSON 5종 | `data/pipeline/01_workflow_s3_scan.json` 외 4개 |
| 파이썬 스크립트 6종 | `data/pipeline/` (`s3_scan.py`, `clean_encoding.py`, `validate_json.py`, `embed_pipeline.py`, `build_weather_rules.py`, `build_combination_rules.py`) |
| docker 설정 | `data/pipeline/n8n/docker-compose.yml`, `data/pipeline/n8n/Dockerfile` |
| 원본 설계 README | `data/pipeline/n8n/README.md` |

주의: n8n에 임포트된 워크플로의 Execute Command 노드는 `python3 /repo/data/pipeline/...` 경로를 참조한다.
docker-compose가 레포 루트를 `/repo`로 RW 마운트하므로([근거: docker-compose.yml `../../..:/repo:rw`]), 스크립트는 `data/pipeline/`에 둔다. [근거: docker-compose.yml volumes, workflow JSON command]

## 5. 실행 방법

전제: Docker Desktop 실행 중, Infisical CLI 로그인 완료.

```powershell
# 1. compose 폴더로 이동 (반드시 이 폴더 — /repo 마운트 기준점)
cd <repo>\data\pipeline\n8n

# 2. 비밀값 주입해 기동 (첫 실행은 이미지 빌드로 수 분 소요)
infisical run --env=dev -- docker compose up -d --build

# 3. 로그 확인 — "Editor is now accessible" 뜨면 기동 완료
docker compose logs -f n8n

# 4. 중지
docker compose down          # 워크플로는 named volume(n8n_data)에 유지
```

- 접속: http://localhost:5678 → 첫 접속 시 오너 계정 생성 (n8n 2.x는 basic auth 제거). [근거: n8n/README.md, docker-compose.yml]
- 워크플로 5종 임포트 완료. 01~03(ID: `WORKFLOW_ID_SCAN` / `_CLEAN` / `_EMBED`, 상호 참조 연결 확인됨, **현재 비활성**). 04(`WORKFLOW_ID_WEATHER_RULES`, /rule/weather-build)·05(`WORKFLOW_ID_COMBINATION_BUILD`, /rule/combination-build)는 **2026-07-13 활성화됨** [근거: workflow_04/05 `active: true`].
