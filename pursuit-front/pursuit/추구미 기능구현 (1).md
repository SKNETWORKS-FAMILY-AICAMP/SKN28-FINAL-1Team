## 1. 기능 목표

마이페이지의 `추구미/선호도` 버튼에서 사용자의 스타일 선호 정보를 조회하고 저장할 수 있게 한다.

- 사용자는 선호하는 요소와 피하고 싶은 요소를 토글 버튼을 통해 구분하여 설정한다.
- 모든 항목은 선택하지 않아도 저장할 수 있다.
- 저장된 값은 이후 다시 조회되어 선택 상태(선호/기피 색상 구분)로 보여야 한다.
- 사용자는 언제든 마이페이지에서 기존 선택값을 수정할 수 있어야 한다.
- 옵션 목록은 백엔드에서 통합 관리하며, 선호/기피 섹션에서 동일한 옵션 풀을 공유한다.

## 2. 현재 확정 요구사항

- **토글 UI 도입**: 상단에 `선호해요` (Blue)와 `피하고 싶어요` (Red) 버튼을 두어 선택 모드를 전환한다.
- **통합 옵션 풀**: 모든 카테고리(스타일, 색상, 핏 등)는 선호/기피 모드에서 동일한 목록을 보여준다.
- **최신 선택 우선 정책**: 동일한 항목을 선호와 기피 양쪽에서 중복 선택할 경우, 가장 나중에 선택한 상태를 최종값으로 반영한다. (예: 선호 '그레이' 선택 후 기피 '그레이' 선택 시 기피로 확정)
- **시각적 강화**: 색상 칩에는 컬러 박스를, 핏/기장 등에는 아이콘을 포함하여 직관성을 높인다.
- **Optional 저장**: 모든 항목은 선택 사항이며, 빈 상태로도 저장 가능하다.
- **레이아웃 최적화**: 모든 칩(버튼)의 시작점을 왼쪽으로 맞추고, 불필요한 여백 없이 촘촘하게 배치한다. 계절 섹션 우측의 'v' 아이콘을 삭제한다.

## 4. UI 흐름 및 상태 로직

1. 사용자가 화면 진입 시 옵션 목록과 기존 저장값을 호출한다.
2. `currentMode` 기본값은 `preferred`(선호)로 설정된다.
3. 사용자가 칩을 클릭할 때:
   - 해당 옵션이 이미 반대 모드(기피)에 있다면 기피 목록에서 제거하고 선호 목록에 추가한다. (최신 선택 우선)
   - 이미 같은 모드에 있다면 선택을 해제한다.
4. 토글 버튼 클릭 시 `currentMode`만 변경되며, 화면의 모든 선택 상태(Blue/Red 하이라이트)는 유지된다.
5. `저장하기` 클릭 시 최종 `preferred` 및 `avoided` 객체를 서버로 전송한다.
